## 수정 내용

### 1. 04-프로필편집.html 생성

**파일 경로**: `design/uiux/prototype/04-프로필편집.html`

**주요 기능**:
- 사용자 프로필 정보 편집
- 아바타 변경 버튼 (UI only)
- 기본 정보 폼:
  - 이름
  - 이메일
  - 휴대폰 번호
- 가족 정보 폼:
  - 결혼 여부
  - 자녀 수
  - 현재 거주 형태
  - 연소득 구간
- LocalStorage 연동으로 데이터 저장/로드

**네비게이션**:
- 진입: 00-대시보드.html > 더보기 메뉴 > 프로필 편집
- 뒤로 가기: history.back()

---

### 2. 16-계산결과목록.html 생성

**파일 경로**: `design/uiux/prototype/16-계산결과목록.html`

**주요 기능**:
- 저장된 계산 결과 목록 조회
- 필터 기능:
  - 전체
  - 최근 7일
  - 이번 달
- 각 계산 결과 카드 표시:
  - 주택 정보
  - 대출상품
  - 매매가/대출액
  - 월 총 지출
  - LTV/DTI/DSR 지표 (색상 코딩)
- 계산 결과 삭제 기능
- 빈 상태 UI
- 샘플 데이터 2개 자동 생성

**네비게이션**:
- 진입: 00-대시보드.html > 더보기 메뉴 > 계산 결과 목록
- 카드 클릭: 15-입주후지출계산.html로 이동
- 새 계산하기 버튼: 15-입주후지출계산.html로 이동

### 3. 02-회원가입.html 수정

**변경 사항**:
```javascript
// 변경 전 (line 605)
window.location.href = '01-로그인.html';

// 변경 후
window.location.href = '03-기본정보입력.html';
```

**수정 이유**:
- 회원가입 완료 후 로그인 화면으로 돌아가는 것은 UX상 부자연스러움
- 새 사용자는 회원가입 후 바로 기본 정보 입력 단계로 진행하는 것이 자연스러운 플로우

**업데이트된 플로우**:
```
01-로그인.html
→ (회원가입 링크 클릭)
→ 02-회원가입.html
→ (회원가입 완료)
→ 03-기본정보입력.html
→ 05-본인자산정보입력.html
→ 06-배우자자산정보입력.html
→ 00-대시보드.html
```

## LocalStorage 키 목록

### 사용자 정보
- `auth_token`: 인증 토큰
- `user_info`: 사용자 기본 정보

### 자산 정보
- `self_asset`: 본인 자산 데이터
- `spouse_asset`: 배우자 자산 데이터
- `assets`: 본인 자산 목록
- `loans`: 본인 대출 목록
- `income`: 본인 월소득 목록
- `expense`: 본인 월지출 목록
- `spouse_assets`: 배우자 자산 목록
- `spouse_loans`: 배우자 대출 목록
- `spouse_income`: 배우자 월소득 목록
- `spouse_expense`: 배우자 월지출 목록

### 대출 정보
- `loan_products`: 대출상품 목록
- `loan_bookmarks`: 북마크한 대출상품 ID 목록
- `selected_loan_id`: 선택된 대출상품 ID
- `selected_loan_for_calc`: 계산에 사용할 대출상품 정보

### 주택 정보
- `housing_list`: 입주희망주택 목록
- `selected_housing_id`: 선택된 주택 ID

### 계산 정보
- `calculations`: 계산 결과 목록
- `selected_calculation`: 선택된 계산 결과

### 로드맵 정보
- `lifecycle_events`: 생애주기 이벤트 목록

## 테스트 방법

### 1. 브라우저에서 수동 테스트
```bash
# 프로토타입 디렉토리에서
open 01-로그인.html
# 또는
open 00-대시보드.html
```

### 2. 온보딩 플로우 테스트
1. 01-로그인.html 열기
2. "회원가입" 링크 클릭
3. 02-회원가입.html에서 정보 입력 후 가입
4. 자동으로 03-기본정보입력.html로 이동하는지 확인
5. 기본 정보 입력 후 "다음" 클릭
6. 05-본인자산정보입력.html로 이동하는지 확인
7. 전체 플로우 완료 후 00-대시보드.html 도달 확인

### 3. 신규 화면 테스트

**04-프로필편집.html**:
1. 대시보드 하단 탭의 "더보기" 버튼 클릭
2. "프로필 편집" 메뉴 클릭
3. 프로필 정보 확인 및 수정
4. "저장" 버튼 클릭
5. 토스트 메시지 확인
6. 대시보드로 돌아가는지 확인

**16-계산결과목록.html**:
1. 대시보드 "더보기" 메뉴에서 "계산 결과 목록" 클릭
2. 샘플 계산 결과 2개 확인
3. 필터 기능 테스트 (전체/최근 7일/이번 달)
4. 계산 결과 카드 클릭하여 15번 화면으로 이동 확인
5. 삭제 버튼 클릭하여 삭제 기능 확인
6. "새 계산하기" 버튼 클릭하여 15번 화면으로 이동 확인

### 4. 모든 네비게이션 링크 테스트
- 대시보드의 모든 링크 클릭 테스트
- 하단 탭 네비게이션 4개 모두 클릭 테스트
- 각 화면의 뒤로 가기 버튼 테스트
- JavaScript 네비게이션 (버튼 클릭) 테스트

## 알려진 제한사항

1. **실제 인증 없음**: 모든 인증은 LocalStorage 기반 시뮬레이션
2. **백엔드 API 없음**: 모든 데이터는 클라이언트에만 저장
3. **파일 업로드 미구현**: 아바타 변경은 UI만 제공
4. **04번 화면 누락**: 대출인증.html은 프로토타입에서 생략됨 (03번에서 바로 05번으로)
