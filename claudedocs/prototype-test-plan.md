# 프로토타입 테스트 계획서

## 목적
내집마련 도우미 플랫폼 HTML 프로토타입의 기능 및 UI/UX 검증

## 테스트 범위
전체 18개 화면 (01-18번) + 관리자 화면 (10번)

---

## 1. 기본 기능 테스트

### 1.1 온보딩 플로우 테스트

**테스트 시나리오**: 신규 사용자 회원가입 및 기본 정보 입력

**테스트 단계**:
1. `01-로그인.html` 접속
   - "회원가입" 버튼 클릭
2. `02-회원가입.html`에서 입력
   - 이름: "홍길동"
   - 이메일: "test@example.com"
   - 비밀번호: "Test1234!"
   - 비밀번호 확인: "Test1234!"
   - "회원가입" 버튼 클릭
3. `03-기본정보입력.html`에서 입력
   - 결혼 여부: "기혼" 선택
   - 자녀 수: "1명" 선택
   - 현재 거주 형태: "전세" 선택
   - 소득 수준: "5000만원 이상 ~ 7000만원 미만" 선택
   - "다음" 버튼 클릭
4. `04-대출인증.html` 확인
   - "다음에 하기" 또는 "인증하기" 클릭

**예상 결과**:
- 각 화면 전환이 원활하게 진행됨
- 입력 데이터가 LocalStorage에 저장됨
- 유효성 검사 동작 확인 (이메일 형식, 비밀번호 조건 등)

---

### 1.2 자산 정보 입력 테스트

**테스트 시나리오**: 본인 및 배우자 자산 정보 입력

**테스트 단계**:
1. `05-본인자산정보입력.html` 접속
2. 자산 정보 입력:
   - 예금/적금: 5000만원
   - 주식/펀드: 2000만원
   - 부동산: 0원
3. 대출 정보 입력:
   - 주택담보대출: 0원
   - 신용대출: 1000만원
4. 월 소득 입력:
   - 급여소득: 500만원
5. 월 지출 입력:
   - 생활비: 250만원
   - 대출상환: 50만원
6. "다음" 버튼 클릭
7. `06-배우자자산정보입력.html`에서 동일한 과정 반복
8. "저장하고 계속" 버튼 클릭

**예상 결과**:
- 모든 입력값이 LocalStorage에 정확히 저장됨
- 금액 형식이 올바르게 표시됨 (쉼표 구분)
- 다음 단계로 정상 이동

---

### 1.3 대시보드 및 자산 관리 테스트

**테스트 시나리오**: 대시보드에서 자산 정보 조회 및 관리

**테스트 단계**:
1. `00-대시보드.html` 접속
2. 재무 상태 카드 확인:
   - 순자산 금액 확인
   - 월 가용 소득 확인
3. "자산 정보 수정" 버튼 클릭
4. `07-자산정보관리.html` 접속
5. 본인 탭에서 자산 항목 추가:
   - 카테고리: "자산"
   - 항목명: "청약통장"
   - 금액: "1000"
   - "추가" 버튼 클릭
6. 추가된 항목 수정:
   - 금액을 "1500"으로 변경
   - "저장" 버튼 클릭
7. 항목 삭제:
   - 삭제 버튼 클릭
   - 확인 다이얼로그에서 "확인"
8. 배우자 탭 전환 및 동일 테스트 수행

**예상 결과**:
- 요약 카드에 총계가 정확히 계산되어 표시됨
- CRUD 작업(추가/조회/수정/삭제)이 모두 정상 동작
- 탭 전환 시 데이터가 올바르게 유지됨
- LocalStorage 업데이트 확인

---

### 1.4 대출상품 검색 및 조회 테스트

**테스트 시나리오**: 대출상품 검색, 필터링, 북마크 기능 테스트

**테스트 단계**:
1. `00-대시보드.html`에서 "대출상품 찾기" 버튼 클릭
2. `08-대출상품목록.html` 접속
3. 검색 기능 테스트:
   - 검색어 "신혼" 입력
   - 신혼부부 관련 상품만 표시되는지 확인
   - 검색어 지우기 (X 버튼)
4. 필터 기능 테스트:
   - "신혼부부" 필터 선택
   - "생애최초" 필터 선택
   - "전체" 필터로 복귀
5. 정렬 기능 테스트:
   - "금리순" 선택 → 금리 낮은 순 정렬 확인
   - "한도순" 선택 → 한도 높은 순 정렬 확인
   - "최신순" 선택 → 최신 등록 순 정렬 확인
6. 북마크 기능 테스트:
   - 상품 카드의 별 아이콘 클릭
   - 토스트 메시지 "북마크에 추가되었습니다" 확인
   - 다시 클릭하여 북마크 해제
7. 상품 카드 클릭하여 상세 페이지 이동

**예상 결과**:
- 검색어에 따라 상품이 필터링됨
- 필터 칩 선택 시 해당 카테고리 상품만 표시
- 정렬 순서가 올바르게 적용됨
- 북마크 상태가 LocalStorage에 저장됨
- 상세 페이지로 정상 이동

---

### 1.5 대출상품 상세 조회 및 계산 연동 테스트

**테스트 시나리오**: 대출상품 상세 정보 확인 및 계산기 연동

**테스트 단계**:
1. `09-대출상품상세.html` 접속
2. 헤더 카드 정보 확인:
   - 대출한도, 금리, LTV, DTI 하이라이트 확인
3. 상세 정보 섹션 확인:
   - 기본정보
   - 자격요건 체크리스트
   - 추가정보
   - 신청 방법 (4단계 가이드)
4. 북마크 기능 테스트:
   - 헤더의 별 아이콘 클릭
   - 북마크 토글 확인
5. "이 대출로 계산하기" 버튼 클릭
6. `15-입주후지출계산.html`로 이동 확인
7. 선택된 대출상품 정보가 계산 화면에 표시되는지 확인

**예상 결과**:
- 모든 상세 정보가 정확히 표시됨
- 북마크 상태 변경이 LocalStorage에 반영됨
- 대출상품 선택이 계산 화면으로 전달됨
- 토스트 메시지가 적절히 표시됨

---

### 1.6 관리자 대출상품 관리 테스트

**테스트 시나리오**: 관리자가 대출상품을 추가/수정/삭제/활성화 관리

**테스트 단계**:
1. `10-대출상품관리.html` 접속
2. 통계 카드 확인:
   - 전체 상품 수
   - 활성 상품 수
   - 비활성 상품 수
3. "상품 추가" 버튼 클릭
4. 대출상품 추가 폼 입력:
   - 대출이름: "테스트 대출상품"
   - 대출한도: 30000만원
   - LTV: 70%
   - DTI: 60%
   - DSR: 40%
   - 금리: 3.5%
   - 대상주택: "9억 이하 주택"
   - 대상유형: "일반" 선택
   - 자격요건:
     ```
     무주택 세대주
     연소득 1억원 이하
     ```
   - 활성화: 체크
5. "저장" 버튼 클릭
6. 추가된 상품이 테이블/카드에 표시되는지 확인
7. 수정 기능 테스트:
   - 방금 추가한 상품의 "✏️" 버튼 클릭
   - 금리를 "3.2%"로 변경
   - "저장" 버튼 클릭
8. 활성화/비활성화 테스트:
   - "⏸️" 버튼 클릭하여 비활성화
   - 상태 배지가 "비활성"으로 변경되는지 확인
   - "▶️" 버튼 클릭하여 다시 활성화
9. 삭제 기능 테스트:
   - "🗑️" 버튼 클릭
   - 확인 다이얼로그에서 "확인"
   - 상품이 목록에서 제거되는지 확인
10. 통계 카드 업데이트 확인

**예상 결과**:
- 상품 추가 시 폼 유효성 검사 동작
- 추가된 상품이 즉시 목록에 표시됨
- 수정 내용이 정확히 반영됨
- 활성화 상태 토글이 정상 동작
- 삭제 시 확인 다이얼로그 표시
- 통계가 실시간으로 업데이트됨
- 모든 변경사항이 LocalStorage에 저장됨

---

### 1.7 주택 정보 입력 및 관리 테스트

**테스트 시나리오**: 입주 희망 주택 정보 입력 및 목록 관리

**테스트 단계**:
1. `11-입주희망주택입력-기본정보.html` 접속
2. 기본 정보 입력:
   - 주택 유형: "아파트" 선택
   - 지역: "서울특별시" 선택
   - 상세 주소: "강남구 역삼동"
   - 전용면적: "84" ㎡
   - "다음" 버튼 클릭
3. `12-입주희망주택입력-상세정보.html` 접속
4. 상세 정보 입력:
   - 매매가: 100000만원
   - 희망 입주일: 날짜 선택
   - 보증금: 5000만원
   - 월세: 0원
   - 관리비: 30만원
   - "저장" 버튼 클릭
5. `13-입주희망주택목록.html`로 이동
6. 추가된 주택 카드 확인:
   - 주택 유형, 지역, 면적 표시
   - 매매가, 월세 정보 표시
7. 주택 카드 클릭하여 `14-입주희망주택상세.html` 이동
8. 상세 정보 확인:
   - 모든 입력 정보가 정확히 표시되는지 확인
9. "수정" 버튼 클릭하여 수정 기능 테스트
10. "삭제" 버튼 클릭하여 삭제 기능 테스트

**예상 결과**:
- 2단계 입력 플로우가 정상 동작
- 입력 데이터가 LocalStorage에 저장됨
- 목록에서 카드 형식으로 잘 표시됨
- 상세 페이지에서 모든 정보 확인 가능
- 수정/삭제 기능 정상 동작

### 1.8 입주 후 지출 계산 테스트

**테스트 시나리오**: 대출상품과 주택 정보를 기반으로 월 지출 계산

**테스트 단계**:
1. `15-입주후지출계산.html` 접속
2. 계산 입력 폼 확인:
   - 선택된 대출상품 정보 표시 확인
   - 선택된 주택 정보 표시 확인
3. 계산 조건 입력:
   - 대출 금액: 70000만원
   - 대출 기간: 30년
   - 상환 방식: "원리금균등상환" 선택
4. "계산하기" 버튼 클릭
5. 계산 결과 확인:
   - 월 상환액
   - 월 관리비
   - 월 총 지출
   - 연간 총 지출
   - LTV, DTI, DSR 비율
6. 결과가 합리적인지 검증:
   - LTV = (대출금액 / 주택가격) × 100
   - 월 상환액이 합리적인 범위인지 확인
7. "저장" 버튼 클릭하여 계산 결과 저장
8. "다시 계산" 버튼으로 재계산 테스트

**예상 결과**:
- 대출상품과 주택 정보가 자동으로 불러와짐
- 계산이 정확하게 수행됨
- 결과가 읽기 쉬운 형식으로 표시됨
- 계산 결과를 LocalStorage에 저장 가능
- 재계산 기능 정상 동작

- 이벤트가 정확히 추가/수정/삭제됨
- 로드맵에 이벤트가 타임라인에 표시됨
- 재무 상태 변화가 시각적으로 표현됨
- 다운로드/공유 기능 동작

## 2. 네비게이션 테스트

### 2.1 하단 탭 네비게이션 테스트

**테스트 대상 화면**:
- 00-대시보드.html
- 13-입주희망주택목록.html
- 15-입주후지출계산.html
- 18-장기주거로드맵조회.html

**테스트 단계**:
1. 각 화면에서 하단 탭 4개 확인:
   - 홈 (대시보드)
   - 주택 (입주희망주택목록)
   - 계산 (입주후지출계산)
   - 로드맵 (장기주거로드맵조회)
2. 각 탭 클릭 시 해당 화면으로 이동하는지 확인
3. 현재 화면의 탭이 활성화 상태(active)인지 확인

**예상 결과**:
- 모든 탭 클릭 시 정상 이동
- 활성 탭의 색상과 아이콘이 강조됨
- 탭 레이블이 명확하게 표시됨

---

### 2.2 뒤로 가기 버튼 테스트

**테스트 단계**:
1. 모든 화면에서 헤더의 뒤로 가기 버튼 확인

**예상 결과**:
- 모든 화면에서 뒤로 가기 버튼 정상 동작
- history.back() 동작 확인

3. 링크 텍스트 색상 대비 확인

```javascript
// tests/01-login-flow.spec.js
const { test, expect } = require('@playwright/test');

test('로그인에서 회원가입으로 이동', async ({ page }) => {
  await page.goto('file:///path/to/01-로그인.html');

  // 페이지 제목 확인
  await expect(page.locator('.header-title')).toHaveText('로그인');

  // 회원가입 버튼 클릭
  await page.click('text=회원가입');

  // 회원가입 페이지로 이동 확인
  await expect(page).toHaveURL(/02-회원가입\.html$/);
  await expect(page.locator('.header-title')).toHaveText('회원가입');
});

test('회원가입 폼 유효성 검사', async ({ page }) => {
  await page.goto('file:///path/to/02-회원가입.html');

  // 빈 폼 제출 시도
  await page.click('button[type="submit"]');

  // HTML5 유효성 검사 메시지 확인
  const nameInput = page.locator('#name');
  const isInvalid = await nameInput.evaluate(el => !el.validity.valid);
  expect(isInvalid).toBe(true);

  // 올바른 데이터 입력
  await page.fill('#name', '홍길동');
  await page.fill('#email', 'test@example.com');
  await page.fill('#password', 'Test1234!');
  await page.fill('#confirmPassword', 'Test1234!');

  // 약관 동의
  await page.check('#termsAgree');

  // 제출
  await page.click('button[type="submit"]');

  // 다음 페이지로 이동 확인
  await expect(page).toHaveURL(/03-기본정보입력\.html$/);
});
```

### 대출상품 검색 테스트

```javascript
// tests/08-loan-search.spec.js
const { test, expect } = require('@playwright/test');

test('대출상품 검색 및 필터', async ({ page }) => {
  await page.goto('file:///path/to/08-대출상품목록.html');

  // 초기 상품 개수 확인
  const initialCount = await page.locator('.loan-card').count();
  expect(initialCount).toBeGreaterThan(0);

  // 검색어 입력
  await page.fill('.search-box input', '신혼');

  // 검색 결과 필터링 확인
  const searchResults = await page.locator('.loan-card').count();
  expect(searchResults).toBeLessThanOrEqual(initialCount);

  // 검색어에 "신혼"이 포함되는지 확인
  const loanNames = await page.locator('.loan-name').allTextContents();
  loanNames.forEach(name => {
    expect(name.toLowerCase()).toContain('신혼');
  });

  // 검색어 지우기
  await page.click('.search-clear');

  // 전체 상품 다시 표시 확인
  const afterClearCount = await page.locator('.loan-card').count();
  expect(afterClearCount).toBe(initialCount);
});
```

## 14. 알려진 제한사항

### 프로토타입의 제한사항
1. **인증 기능**: 실제 인증이 구현되지 않음. 데모용 LocalStorage만 사용
2. **API 통합**: 백엔드 API가 없어 모든 데이터가 클라이언트에만 저장됨
3. **결제 기능**: 결제 프로세스는 UI만 제공
4. **파일 업로드**: 파일 업로드 기능은 미구현
5. **푸시 알림**: 알림 기능은 미구현
6. **오프라인 지원**: 오프라인 모드 미지원

### 테스트 시 주의사항
1. LocalStorage는 브라우저별로 분리되어 있어 브라우저 간 데이터 공유 안 됨
2. 시크릿/프라이빗 모드에서는 LocalStorage가 제한될 수 있음
3. 모바일 실제 디바이스에서 테스트 권장 (에뮬레이터는 완벽하지 않음)
