# LLM 활용 분석/기획/설계 진행 가이드

구현될 솔루션을 활용하여 해결하고자 하는 문제와 타겟층을 정의하면
LLM이 정의된 내용을 기반으로 분석부터 설계까지 진행합니다.
진행에 Claude Opus 4.5 모델을 활용하였습니다.

LLM이 도출한 결과물은 사람이 항시 검증하고
개입이 필요한 경우 개입하여 자료를 수정합니다.

각 단계별로 LLM이 활용된 경우 (LLM 활용) 표시, 사람이 수정한 경우 (수정) 표시

### 2. 5why 분석 (LLM 활용)

- 5why 분석 기법을 활용하여 근본원인을 도출
- 프롬프트
```
o: 현상문제과 근본원인을 도출해 주세요.
[요청사항]
- 가장 큰 현상문제 3개를 도출하고 5WHY기법으로 근본문제를 식별.
- {현상문제|WHY1|WHY2|WHY3|WHY4|WHY5|근본원인}형식으로 작성
  - WHY1은 현상문제에 대한 질문 + 질문에 대한 응답으로 작성   
  - WHY2~WHY5는 이전 WHY의 응답에 대한 질문 + 질문에 대한 응답으로 작성 
  - 예)WHY1의 응답이 '선택지가 많기 때문이다.'일때   
    WHY2는 '왜 선택지가 많은가? 다양한 니즈가 혼재되어 있기 때문이다.'와 같이 작성    
- 근본원인을 다층적으로 접근하여 식별
  - 기술적 측면: 시스템 성능, 기능, 안정성, 사용성
  - 사용자 심리적 측면: 인지 부하, 감정, 동기, 습관, 저항감
  - 제도/시스템적 측면: 정책/규정, 프로세스, 권한 구조, 인센티브, 예산/자원
  - 맥락적 측면: 조직 문화, 업계 관행, 시장 상황, 타이밍
  - 커뮤니케이션 측면: 정보 전달, 교육, 피드백 체계
- Sequential MCP 이용 

[참고자료]
- define/문제정의.md
[예시]
- reference/sample_문제가설.md
[결과형식]
- 마크다운 표
[결과파일]
define/문제가설.md
```

```
UI/UX설계서를 작성해 주세요.

아래 순서에 따라 순차적으로 작성

  - 화면 간 사용자 플로우 정의 
  - 화면별 상세 설계: 
    - 개요: 목적, 관련 유저스토리, 비즈니스 중요도 
    - 주요 기능 
    - UI 구성요소
    - 인터랙션
  - 화면간 전환 및 네비게이션 
  - 반응형 설계 전략 
  - 접근성 보장 방안 
  - 성능 최적화 방안 
  - 변경 이력    
- 검토: 
  - <작성원칙> 준수 검토
```
